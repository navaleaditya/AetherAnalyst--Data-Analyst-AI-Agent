

Autonomous Data Analyst AI Agent
Technical Architecture & Stack Documentation

Version 1.0  |  Project Documentation


1. Project Overview
The Autonomous Data Analyst AI Agent is a production-grade, full-stack intelligent system designed to autonomously detect the type of machine learning task from uploaded datasets and execute end-to-end data analysis without requiring manual user intervention. The agent covers classification, clustering, and anomaly detection workflows through an agentic loop architecture.

Key highlights:
Auto-detects ML task type from uploaded data
Executes classification, clustering, and anomaly detection autonomously
94%+ accuracy on test datasets with automated feature importance and SHAP-based explainability
6-tool agentic loop: dataset profiling → correlation analysis → chart generation → sandboxed code execution → session memory
Processes 1,000+ records in under 15 seconds
Drag-and-drop upload, real-time pipeline visualization, and PDF report export

3. Detailed Technology Breakdown
3.1  LangGraph — Agentic Loop Orchestration
LangGraph is the backbone orchestration framework of the entire agent. It models the analysis pipeline as a directed graph of tool nodes, allowing the agent to traverse steps conditionally based on data signals.

What It Is
A graph-based orchestration library built on top of LangChain. It enables stateful, multi-step agent workflows where each node is a callable tool and edges define conditional transitions.
How It Is Used
Defines a 6-node agentic loop: Dataset Profiler → Correlation Analyzer → ML Task Detector → Model Executor → Chart Generator → Session Memory
Manages shared state across tools within a single analysis session
Routes execution path based on the detected ML task type (classification / clustering / anomaly)
Handles tool retries, error edges, and graceful fallback paths
Key Features
Stateful graph execution with persistent in-session memory
Conditional edge routing for dynamic pipeline branching
Native integration with Gemini 2.5 Flash for LLM node calls
Supports sandboxed code execution nodes

3.2  Gemini 2.5 Flash — Core LLM
Gemini 2.5 Flash by Google DeepMind serves as the primary reasoning engine. It is called at decision nodes within the LangGraph pipeline to interpret data characteristics and make natural-language-driven decisions.

What It Is
A multimodal large language model optimized for speed and efficiency, part of Google's Gemini 2.5 family.
How It Is Used
Auto-detects the ML task type (classification vs. clustering vs. anomaly detection) by analyzing dataset schema and statistical summaries
Generates natural language analysis summaries for each pipeline stage
Powers the conversational layer for user-facing explanations of results
Interprets SHAP values and translates them into human-readable feature importance explanations
Key Features
Sub-second inference latency suitable for real-time pipelines
Multimodal input support (text + structured data context)
High instruction-following accuracy for structured JSON output generation

3.3  Auto-ML: KMeans, GradientBoosting & IsolationForest
The agent implements auto-ML selection — it picks and executes the appropriate algorithm based on the detected task type, without any user configuration.

KMeans — Clustering
Used when no target label column is detected in the dataset
Groups records into k clusters based on feature similarity
Outputs cluster assignments and centroid profiles per group
Integrated with Matplotlib to auto-generate cluster scatter plots

GradientBoosting — Classification
Used when a categorical or binary target column is detected
Achieves 94%+ accuracy on benchmark test datasets
Outputs class predictions, probability scores, and classification report
Feature importances extracted and passed to SHAP for explainability

IsolationForest — Anomaly Detection
Used when the task is identified as outlier or anomaly detection
Isolates anomalies by randomly partitioning feature space
Outputs anomaly scores and flags records as normal or anomalous
Results visualized via Matplotlib scatter overlays

3.4  SHAP — Explainability
SHAP (SHapley Additive exPlanations) is integrated into the post-model analysis step to provide transparent, human-interpretable explanations for model predictions.

How It Is Used
Computes per-feature Shapley values after GradientBoosting classification
Generates summary bar charts of global feature importance
Outputs per-prediction local explanations
Results are passed to Gemini 2.5 Flash for natural language narration
Key Features
Model-agnostic explainability compatible with tree-based models
Both global (dataset-wide) and local (per-record) explanations
Visual outputs exported as charts embedded in the PDF report

3.5  React — Frontend Application
The frontend is built with React and deployed on Vercel. It provides the user-facing interface for data upload, real-time pipeline visualization, and report export.

How It Is Used
Drag-and-drop file upload interface for CSV/Excel datasets
Real-time pipeline visualization — each agentic step lights up as it executes
Renders charts and SHAP visualizations returned by the backend
PDF report export button triggers download of the full analysis report
Session state management for multi-step analysis flows
Key Features
Component-based architecture for modular UI elements
Real-time state updates via API polling or WebSocket
Responsive layout compatible with desktop and tablet

3.6  FastAPI — Backend API Server
FastAPI serves as the backend layer, exposing REST endpoints that the React frontend calls. It handles file ingestion, triggers the LangGraph pipeline, and returns structured analysis results.

How It Is Used
POST /upload — accepts dataset file and initiates the agentic pipeline
GET /status/{session_id} — returns real-time pipeline progress
GET /results/{session_id} — returns full analysis output including charts and metrics
GET /report/{session_id} — returns downloadable PDF report
Key Features
Async request handling for non-blocking ML inference
Automatic OpenAPI / Swagger documentation generation
Pydantic-based request validation
Integrated directly with LangGraph agent invocation

3.7  Pandas & NumPy — Data Processing
Pandas and NumPy form the data processing core of the agent's dataset profiling and feature engineering steps.

Pandas
Ingests uploaded CSV/Excel files into DataFrames
Performs dataset profiling: shape, dtypes, null counts, statistical summaries
Executes correlation matrix computation for the Correlation Analyzer tool node
Cleans and prepares features for ML model input

NumPy
Handles numerical array operations underlying ML computations
Used for feature scaling, normalization, and matrix transformations
Supports SHAP value array processing

3.8  Matplotlib — Chart Generation
Matplotlib is the visualization engine inside the agentic pipeline. It runs within sandboxed code execution nodes to generate charts that are embedded in the final report.

How It Is Used
Generates correlation heatmaps during the Correlation Analyzer step
Produces cluster scatter plots for KMeans output
Renders anomaly score distribution histograms for IsolationForest results
Creates SHAP summary bar charts for feature importance
All charts are serialized as PNG and returned via FastAPI to the React frontend
Key Features
Headless rendering (no display required) suitable for server-side execution
High-quality PNG/SVG output for PDF embedding

3.9  Vercel & Render — Deployment

Vercel — Frontend Hosting
Hosts the React application with global CDN distribution
Automatic CI/CD from GitHub repository on push to main
Zero-config HTTPS and custom domain support
Edge network ensures low-latency frontend delivery worldwide

Render — Backend Hosting
Hosts the FastAPI backend as a persistent web service
Supports long-running ML inference requests without timeout issues
Auto-deploys on GitHub pushes
Environment variable management for API keys and config
